# 30/60/90 Day Security Posture Plan

## 30 Days: Quick Wins & Stabilization
* **Automated Regression:** Schedule the `run_all.sh` test suite to run via `cron` nightly and report failures via email.
* **Key Rotation Policy:** Implement a mandatory 90-day rotation cadence for all SSH ED25519 administrative keys.
* **Backup Verification:** Ensure all firewall rulesets and VPN configurations are backed up and restorable within 15 minutes.
* **Patching:** Apply critical OS and service updates to the gateway and web servers.

## 60 Days: Architecture & Visibility Improvements
* **Centralized Log Collection:** Deploy a lightweight forwarder (e.g., Filebeat) on all VMs to send `auth.log`, `suricata/fast.log`, and firewall deny logs to a central SIEM.
* **Web Application Firewall (WAF):** Layer a WAF in front of Nginx to filter SQL injection and Cross-Site Scripting (XSS) before it hits the application logic.
* **Micro-segmentation Review:** Audit the reachability matrix to see if further host-based firewalls (UFW) can restrict intra-DMZ traffic.

## 90 Days: Governance & Zero Trust Automation
* **Infrastructure as Code (IaC):** Migrate manual NFTables and IPsec configurations into Ansible playbooks to prevent configuration drift.
* **PKI Migration:** Replace the strongSwan IPsec Pre-Shared Key (PSK) with X.509 Certificates for scalable, identity-based authentication.
* **Zero Trust Pilots:** Begin testing an identity-aware proxy (e.g., SD-WAN Service Attributes / ZTNA) to eventually replace the traditional VPN tunnel.
